Requires `weclaw_dev >= v0.1.9-alpha` if WeClaw integration is used.

Highlights since v0.3.8-alpha:

- WeClaw status and Details are now dsproxy-owned contracts. `/status`, `--weclaw-json`, and local status endpoints expose profile, model, effort, session, context window, token usage, prompt cache, pricing, estimated cost, provider balance, diagnostics, and action hints directly from dsproxy, so WeClaw does not need to infer or recompute them.
- The current-session token and cost accounting contract is explicit. Last-turn and session totals are separated from route/global totals; auxiliary model calls report a clear zero object when absent; provider usage remains authoritative for billing, prompt cache hit/miss accounting, and request-level totals.
- Details now explains prompt-token origin instead of dumping unexplained totals. It separates user input, history, tool output, system/developer/environment material, compaction summaries, tools schema, protocol overhead, and provider residual, with `details_origin_breakdown` and prompt-reconciliation fields for display.
- The model context window is reported as capacity, not as a compact trigger. Managed DeepSeek V4 profiles keep `model_context_window_tokens = 1000000` as the display denominator, while the compact trigger is derived from the single managed knob `auto_compact_ratio = 0.90`.
- Codex profile compatibility was repaired for newer Codex releases. Managed `deepseek` and `deepseek-thinking` profiles stay aligned with the selected DeepSeek model/provider and avoid display or runtime drift after Codex upgrades.
- Token-first Compact and TRIM replace the old character-driven control plane for user-visible runtime behavior. Compact/Trim status now reports token estimates, trigger thresholds, targets, before/after/removed tokens, retention metadata, reasons, and observed timestamps.
- Runtime payload reports are persisted and restored. Compact and TRIM reports survive proxy restart through SQLite persistence, so status no longer depends only on process-local memory or debug files; in other words, runtime payload reports are persisted for status recovery.
- The semantic payload compaction path was hardened for real traffic. Old low-risk flattened pytest-success transcripts can be compacted through the real `/v1/responses` thinking route with canary/invariant gates, token-savings metadata, type/risk/action counts, and fallback-to-original behavior.
- Successful pytest output containing negated phrases such as `no failures`, `no traceback`, `no diff`, or `no patch` is no longer misclassified as failure output; only positive failure evidence keeps a payload out of low-risk mutation.
- Payload safety was expanded without exposing raw material. Type-aware handling covers logs, diffs, JSON, pytest output, tracebacks, tool payloads, old text, and image envelopes; first images and protected static blocks are preserved; raw prompts, raw tool arguments, and raw image content remain redacted from normal status surfaces.
- Compact audit and Codex-native local prompt alignment are exposed as redacted metadata. dsproxy records prompt/source fingerprints, retained-recent policy, material classification, and local Compact prompt alignment without publishing raw prompt text.
- DeepSeek pricing and cost display are CNY-first and dsproxy-owned. Pricing metadata, discount state, refresh state, provider usage, and session cost ledger fields are exposed for clients without requiring WeClaw to scrape or recalculate pricing.
- Upgrade and installation behavior is commit-aware. `dsproxy upgrade` skips reinstall only when both public version and release/tag commit match; a same-version commit-aware upgrade detects a changed commit behind the same public version and performs an overwrite install; `--force` and `--force-reinstall` remain explicit reinstall paths.
- Release assets remain `bootstrap.sh` and `install.sh`.
